# reports/

Maintainer log of periodic improvement sessions run against this repo.

These files are not shipped with the npm package and are not part of the public API. They exist so contributors can see what was changed, why, and what the measured impact was (e.g. test pass-rate deltas, Windows compatibility fixes).

If you are a beginner looking for how to **use** continuous-improvement, start with the project [README](../README.md) or [QUICKSTART](../QUICKSTART.md) instead.

## Contents

- `daily-improvement.md` — most recent maintenance log
- `assets/` — screenshots and supporting artifacts
